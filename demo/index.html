The st.eeze front-end development starter kit.

by @cousinwil

Style + Ease = St.eeze

St.eeze is a front-end starter kit written by Wil Everts (get in touch: Twitter, Google, LinkedIn). Wil has been writing HTML since 1994, professionally since 1996, and is best known for his elegant, extensible, and pragmatic UI development practices.

This is not a new CSS framework or or HTML5 reset stylesheet. This is a common starting point for front-end application development. Templates like this are all over the place and are a great way for front-end unicorns to share the way they work with one another. St.eeze has the basic HTML5, CSS, and javascript building blocks you need to start a project on the right foot, but you should definitely tweak things, customize functions, and make it your own.

When it's time to start a new project you can just clone the git repo or pull the files from a folder on your laptop, whatever works for you. This includes a great deal of derivative work, cited as best I could remember below.

Benefits

  • All the pop and sizzle of HTML5 and CSS3 newness!
  • Cross-browser HTML5 support, conditional “ie” body class for css hacking, and a js fix to make IE 5.5-8 behave like IE9.
  • Common CSS in one GET like the html5doctor reset and the .clearfix float container.
  • A simple Object Oriented javascript template including useful design patterns for view-specific event handler inits, simple jQuery ajax & form validation techniques demonstrated, and Google Analytics for js events.
  • A small collection of handy CSS3 classes for round corners, background gradients, and drop shadows.
  • A Model-View PhP framework for dirty/rapid prototyping.
  • A Ruby on Rails application with a sample layout and template.
